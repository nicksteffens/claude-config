- Vite `allowedHosts` blocks requests with tunnel hostname in Host header
- `express-http-proxy` stream piping incompatible with SocketProxy's FakeSocket

The `reverse_proxy` approach, the `VITE_TUNNEL_URL` env var, IPv4 connectivity — all needed to be discovered through real debugging. The winning `vite.html.erb` approach was found by accident while eliminating what didn't work.

### Session 2 Overview
**Duration:** 1 hour
**Main Objective:** Investigate and fix EU-only stale content in Studio tunnel mode (sc-190095)
**Success Rating:** 10/10

### What We Accomplished
- Traced the full request path across capturama, CLI, and front-end repos to understand how tunnel requests flow
- Discovered capturama's `canProxyExternalUrl()` excludes `.movabledev.com` but NOT `.movabledev-eu.com` — EU tunnel traffic routes through the caching http_proxy while US bypasses it
- Added `Cache-Control: no-cache, no-store, must-revalidate` headers to CLI's Express server (`studio-router.js`)
- Added `awaitWriteFinish` debounce to CLI's file watcher (`file-watcher.js`) to prevent reading mid-build
- Verified fix locally — headers confirmed via curl, tested rapid rebuilds
- Boss verified the fix; PR created on CLI repo
- Created Notion page documenting the full investigation and root cause
- Updated Notion with the `.movabledev-eu.com` exclusion gap finding

### Challenges Encountered
- Multiple red herrings — WebKit memory cache, file watcher race conditions, browser cache all looked plausible before isolating the real cause
- Cross-repo complexity — had to trace through capturama (C++), CLI (Node), and front-end (Ember) to understand the full picture

### Most Valuable Collaboration
Systematic root cause analysis — methodically ruled out hypotheses (http_proxy exclusions, WebKit cache, build race conditions, browser cache) until isolating that the EU tunnel domain wasn't excluded from capturama's proxy routing.

### Key Insight
`.movabledev-eu.com` was not in capturama's proxy exclusion list while `.movabledev.com` was. This single missing domain caused all EU tunnel traffic to route through the caching http_proxy, while US traffic correctly bypassed it.

**Human:** Directing — guided investigation, tested locally, provided logs and browser screenshots
